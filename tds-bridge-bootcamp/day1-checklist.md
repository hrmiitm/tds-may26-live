# Day 1 Post-Session Checklist
## TDS Bridge Bootcamp — Setup Day

- [ ] I can open a Linux terminal and recognize the shell prompt (`user@machine:~$`)
- [ ] I can run `python --version`, `git --version`, and `uv --version` without errors
- [ ] I can open VS Code connected to WSL/Linux using `code .`
- [ ] I know what `~` means and can navigate to my home directory using `cd ~`
- [ ] I can create folders and files using `mkdir` and `touch`, and list them with `ls`
- [ ] I know the difference between `.xyz`, `./xyz`, and `../xyz`
- [ ] I know that in Linux, everything is a file
- [ ] I can navigate to the C and D drives within WSL
- [ ] I understand the difference between `>` (overwrite) and `>>` (append)
- [ ] I have a GitHub account and have created the `tds-bootcamp` repository

---

### 📌 Submission Task

1. Go to [github.com](https://github.com) and create the **public** repository named `tds-bootcamp` (if you haven’t already).
2. Inside `tds-bootcamp`, create a file named `day1.md` using the GitHub UI (click **Add file → Create new file**).
3. In `day1.md`, paste:
   - This checklist (copy it from the course page), with each box marked as `[x]` or `[ ]` honestly
   - A short **My Notes** section below it — write 2–3 lines about what you learned, what confused you, or what you want to explore more
4. Commit the file directly to `main` with the message: `day1: add checklist and notes`
5. Paste your repo URL in the submission form (or share it with the instructor).
